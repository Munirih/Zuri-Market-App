# Zuri Market e-commerce Platform Deployment to Kubernetes (K3s)

## 1. Frontend Overview

The frontend is the React frontend for Zuri Market. It displays products fetched from the backend API, supports category filtering, and includes a cart with quantity management.  

## 2. Tech Stack

 Frontend: React, Vite

 DevOps: Nginx, Docker, Docker Hub, K3s (Kubernetes), Terraform, AWS Secrets Manager, AWS EC2, AWS VPC, AWS IAM,  Amazon S3 (with S3 Lockfile), GitHub Actions, Trivy.

## 6. Building for Production

Running Vite server in Kubernetes production environment is not practical because it is designed for local development. Therefore frontend docker image is built in two stages:
1. Build Stage
   
 `npm run build` compiles the vite files and outputs a `dist/` directory with highly optimized, minified static files.
2. Serve with a dedicated web server
   
The packaged static files are served by a highly performance HTTP server like Nginx.

## Docker 

Edit the Dockerfile used to build a docker image locally.

```
# Step 1: Build the Vite files
FROM node:24-alpine AS builder

WORKDIR /app

COPY package*.json ./

RUN npm install

COPY . .

RUN npm run build

# Step 2: Serve using Nginx
FROM nginx:stable-alpine

COPY nginx.conf /etc/nginx/conf.d/default.conf
COPY --from=builder /app/dist /usr/share/nginx/html

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

A custom nginx configuration file is added into the frontend folder to serve the static files from /usr/share/nginx/html and forwards all /api/* requests to the backend Kubernetes service.

```
server {
    listen 80;

    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri /index.html;
    }

    location /api/ {
        proxy_pass http://backend-service:5000/api/;
    }
}
```

`COPY nginx.conf /etc/nginx/conf.d/default.conf` copies the custom Nginx configuration file into the Nginx container. This effectively replaces the default Nginx configuration with the custom one.

`COPY --from=builder /app/dist /usr/share/nginx/html` copies the `dist/` files generated by Vite from the builder stage into the Nginx web root directory.

## 8. Components Reference

Frontend Components

The frontend is composed of reusable React components that work together to provide the shopping experience.

| Component	| Purpose|	Data Received |
|--------|------------|--------|
| Navbar	| Displays the store name and shopping cart button.	| storeName, cartCount, onCartOpen |
| Hero	| Displays the application's hero/banner section. | 	storeName |
| FilterBar	| Allows users to filter products by category.	|activeCategory, onCategoryChange |
| ProductGrid	| Displays the list of products returned from the backend API. Handles loading and error states. |	products, loading, error, onAddToCart |
| CartSidebar	| Displays cart contents, quantities, totals, and cart actions.	| cartItems, cartTotal, onRemove, onUpdateQuantity, onClear, onClose |
| Custom | Hooks |
| Hook	| Purpose	| Returned Data |
| useCart	| Manages shopping cart state and business logic.	| cartItems, addToCart, removeFromCart, updateQuantity, clearCart, cartCount, cartTotal |
